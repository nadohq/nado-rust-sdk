# Security Policy

## Reporting a Vulnerability

The Nado team takes security seriously. If you believe you have found a security vulnerability in this SDK, the Nado smart contracts, or the Nado web app, **please do not open a public GitHub issue.**

All security reports should be submitted through Nado's official bug bounty programs on HackenProof:

---

## Bug Bounty Programs

Nado runs two active bug bounty programs on [HackenProof](https://hackenproof.com/companies/nado):

### 🔐 Smart Contracts — up to $500,000
**Scope:** On-chain contracts in the [`nado-contracts`](https://github.com/nadohq/nado-contracts) repository, deployed on Ink L2.

Examples of in-scope vulnerabilities:
- Theft or loss of funds
- Unauthorized transactions
- Transaction manipulation
- Logic errors (behaviour differs from the business description)
- Reentrancy attacks
- Integer overflows or underflows

👉 [Submit a smart contract report](https://hackenproof.com/programs/nado-smart-contracts)

---

### 🌐 Nado Web — up to $10,000
**Scope:** The Nado web application and frontend.

👉 [Submit a web report](https://hackenproof.com/programs/nado-web)

---

## Eligibility

To qualify for a reward you must:

- Be the **first reporter** of the vulnerability
- Report exclusively through [hackenproof.com](https://hackenproof.com/companies/nado) within 24 hours of discovery
- Provide a **clear description** with steps to reproduce, and attach screenshots or proof-of-concept code where applicable
- Not be a current or former employee or contractor of Nado

See each program page for the full rules and reward tiers.

---

## Responsible Disclosure

- **Do not** publicly disclose any vulnerability before it has been resolved
- **Do not** use automated scanners in a way that generates excessive traffic or degrades service
- **Do not** attempt to access, modify, or delete data that does not belong to you

We appreciate responsible security research and will acknowledge valid reports. Thank you for helping keep Nado and its users safe.
